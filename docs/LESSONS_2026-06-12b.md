## 1. 클라이언트-측 오버레이는 driver.py 로 못 본다 — 하지만 **pilot + `app.view._cells` 덤프가 실 클라 합성 검증**이다

- 메모리에 "달력/시계 오버레이는 driver 검증불가"로 박혀 있는데, 절반만 맞다.
  `driver.py` 가 못 보는 이유는 분명하다: 오버레이는 **서버가 모르는 순수 클라 기능**
  (`app.calendar_panes`)이고 클라 `_composite` 의 `client_overlay` 훅에서만 그려지는데,
  driver 는 서버 렌더 패널만 합성하는 별도 경량 컴포지터라 플러그인 훅을 안 돈다.
- **하지만 검증불가가 아니다**: 테스트가 쓰는 그 경로 — `app.run_test(size=...)` 로 진짜
  Textual App 을 띄우면 **진짜 `_composite` + 진짜 plugin `client_overlay` + 진짜
  `on_mouse_down`** 이 다 돈다. `app.view._cells`(최종 합성 셀 그리드)를 글자만 뽑아
  덤프하면 ‹ 2026-06 › 달력이 그대로 보이고, `app.view.on_mouse_down(_FakeMouse(x,y))`
  로 클릭해 다음 달로 바뀌는 걸 텍스트로 확인할 수 있다.
- **방법**(재사용): 임시 스크립트에서 `sys.path.insert(0,"tests")`→`from harness import
  make_app, server_only, teardown`→`async with app.run_test(size=(44,16)) as pilot`,
  `set_calendar`/`on_mouse_down` 호출 후 `"".join(c[0] or " " for c in row)` 로 행 덤프.
  pixel 이 진짜 필요한 건 폰트 합자뿐 — **배치·존 좌표·클릭 동작은 헤드리스로 다 닫힌다.**
  "driver 안 보임 = 검증 불가" 로 미루지 말 것.

## 2. 기존 "클릭=닫기" 오버레이 위에 새 클릭존을 얹으면 **닫기보다 먼저** hit-test 해야 한다

- 달력 패널은 이미 *아무 데나 클릭=오버레이 닫기*(`clientwidgets.on_mouse_down` 의
  `_close_overlay` 블록)였다. ‹/› 클릭존을 그 블록 **뒤에** 두면 화살표를 눌러도 곧
  닫혀 month-nav 가 절대 안 먹는다 — 같은 좌표가 두 동작에 매핑되니 **순서가 전부**다.
- **해결**: nav-zone 루프를 `_close_overlay` 블록 **직전**에 삽입(우클릭/ctrl/shift-drag
  특수처리 다음). 존 hit 면 `calendar_nav(pid,delta)` 후 `event.stop(); return` 으로
  닫기 경로에 도달조차 안 하게. 본문(화살표 밖) 클릭은 루프를 빠져 기존 닫기로 흐른다.
- 일반화: 클릭존을 추가할 땐 **그 좌표를 이미 소비하던 더 넓은 핸들러가 있는지** 보고,
  좁은(특수) 존을 항상 위에 둔다. 기존 prompt-history/perm/remote 존이 패스스루보다
  앞서는 것과 같은 원리.

## 3. 관찰 전용 훅에 소비 의미를 얹지 말고 **소비용 훅을 따로** 판다

- 키를 가로채야 했지만 기존 `client_key` 는 **관찰 전용**(반환값 무시, ime-indicator 가
  한/영 추정만)이라 여기에 "소비하면 패널로 안 보냄"을 얹으면 의미가 둘로 갈린다.
- **해결**: `client_overlay_key(app,event)->bool` 을 Registry 에 신설 — 소비한 플러그인이
  있으면 True 면 코어가 `prevent_default/stop/return`. `client_key`(관찰)는 그대로 두고
  코어 normal 모드에서 **소비 훅을 먼저, 관찰+send_input 을 나중에** 호출.
- delete-to-disable 보존: 코어는 `self.plugins.client_overlay_key(...)` 결과만 보고
  플러그인 부재 시 빈 루프→False→일반 입력 경로. 클릭존도 `getattr(app,
  "_calendar_nav_zones", {})`·`getattr(app, "calendar_nav", None)` 가드라 calendar
  디렉토리를 지우면 키/클릭 네비게이션이 흔적 없이 사라진다. **계약 테스트에 no-op 한 줄**
  (`reg.client_overlay_key(None,None) is False`)을 같은 CL 에 추가해 회귀를 막았다.

  ([[shared-workspace-parallel-wip]] 의 새 양상: marker 충돌이 아니라 *커밋 혼입*.)
