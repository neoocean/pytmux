본격적으로 다루기 전에, pytmux 가 무엇이고 어떤 구조로 되어 있는지 30초면 잡히는 큰 그림을 먼저 봅니다.

pytmux 는 무엇인가

pytmux 는 Python/Textual 로 만든 tmux 유사 터미널 멀티플렉서입니다. 하나의 터미널 안에서 여러 셸을 패널로 나눠 쓰고, 앱이나 터미널 창을 닫아도 셸 세션이 계속 살아 있게 해 줍니다. 단일 서버(데몬)–다중 클라이언트 구조이며 macOS · Linux · Windows(ConPTY) 를 네이티브로 지원합니다 — WSL·Cygwin 이 필요 없습니다.

tmux 명령어를 다 외우지 못해도 됩니다. 마우스 · 메뉴 · 명령 프롬프트 세 가지 방식 어느 것으로도 거의 모든 동작을 할 수 있습니다.

계층: 탭 → 윈도우 → 패널

pytmux 는 단일 세션 모델입니다. tmux 의 멀티 세션 개념은 없습니다. 항상 하나의 세션으로 시작하고, 탭으로 여러 윈도우를 열며, 각 윈도우를 패널로 나눕니다.

  • 탭 — 최상위. 1번부터 번호가 매겨지고 상단 탭바에 나타납니다.
  • 윈도우 — 각 탭에 종속된 단일 윈도우.
  • 패널 — 윈도우를 분할한 셸 집합. 활성 패널은 파란 테두리, 비활성은 회색.

셸 영속 — 닫아도 살아 있습니다

셸 PTY 를 백그라운드 데몬(서버)이 보유합니다. 그래서 앱을 닫거나(detach) 상위 터미널 창을 닫아도, 실행 중이던 셸·프로그램은 계속 돌아갑니다. 다시 실행하면 끊긴 지점에서 그대로 이어 붙습니다. SSH 로 원격 접속해 작업하다 연결이 끊겨도 세션이 날아가지 않는 것이 tmux 계열의 핵심 가치이고, pytmux 도 이를 그대로 제공합니다.
