마우스로 제어

거의 모든 동작을 마우스로 할 수 있습니다. 이 챕터는 pytmux 의 모든 마우스 제스처를 한자리에 모았습니다. 마우스는 set mouse on(기본값)일 때 동작합니다.

패널(콘텐츠) 영역

제스처동작
왼쪽 클릭패널 포커스(마우스 쓰는 앱이면 앱으로 전달)
왼쪽 드래그텍스트 선택 → OS 클립보드 자동 복사(mouse-drag-copy on, 시작한 패널 안으로만 가둠)
Shift + 왼쪽 드래그텍스트 선택 → 복사(마우스 쓰는 앱 위에서도 강제로 선택)
오른쪽 클릭컨텍스트 메뉴(먼저 그 패널을 활성화). 앱의 마우스 모드보다 우선
휠 ↑ / ↓스크롤백(마우스 쓰는 앱이면 앱으로 전달)
경계선 드래그패널 크기 조절
패널 헤더 드래그패널을 집어 다른 패널에 놓으면 swap, 다른 탭에 놓으면 이동, [+]에 놓으면 새 탭
활성 패널 우측 상단 [x] 클릭탭 닫기(확인) — 패널 닫기는 우클릭 메뉴 패널 삭제 ✕
클릭과 드래그 구분: pytmux 는 누름(press)을 실제 이동(move)이 있을 때까지 미뤄, 클릭은 앱의 버튼/포커스로, 드래그는 선택으로 보냅니다. 그래서 Claude 같은 마우스 앱 위에서도 버튼은 눌리고 드래그는 복사됩니다.

드래그 복사, 자세히

기본값 mouse-drag-copy on 에서, normal 모드의 왼쪽 드래그는 pytmux 가 패널 경계로 클램프한 선택이 되어 손을 떼는 순간 OS 클립보드로 자동 복사됩니다. 이렇게 하면 호스트 터미널의 네이티브 선택이 화면 전체를 긁어 패널 외곽선(│)이나 옆 패널까지 복사되던 불편이 사라집니다.

  • Shift·copy-mode·Ctrl+C 없이 그냥 드래그하면 복사됩니다.
  • 화면 폭 때문에 여러 줄로 접힌 긴 줄(자동 줄바꿈)은 한 줄로 복사됩니다 — 줄바꿈은 실제 줄 끝에만 들어가, 여러 줄에 걸쳐 보이는 긴 명령도 붙여넣으면 그대로 실행됩니다 (Windows 패널 포함).
  • 선택은 드래그를 시작한 패널 안으로만 갇힙니다.
  • Claude 등 마우스 앱 위에서도 드래그는 선택에 양보합니다(클릭은 앱으로).
  • 앱에 드래그를 넘기려면 set mouse-drag-copy off — 그때 선택은 Shift+드래그로.

상태표시줄 · 오버레이 클릭

  • 모델 배지 클릭 → 모델/컨텍스트 변경 팝업
  • 토큰/사용량 배지 클릭 → 토큰 사용량 팝업
  • 권한모드 footer 클릭 → auto / accept / default / plan 선택
  • 서버 이름(host) 클릭 → 서버 정보 탭
  • "esc to interrupt"(처리중 footer) 클릭 → 그 패널에 ESC 주입
  • 시계 클릭 → 시계 모드, 날짜 클릭 → 달력 오버레이
  • 시계·달력 오버레이 클릭 → 오버레이 닫기(달력은 ‹ › 로 월 이동)
모든 제스처는 mouse-help(별칭 mouse) 또는 list-keys 팝업, 그리고 컨텍스트 메뉴의 "마우스 제스처 도움말"에서 언제든 확인할 수 있습니다. Windows 에서 앱에 마우스 이동까지 넘기려면 win-mouse-motion on (기본 off — ConPTY 누출 방지).
