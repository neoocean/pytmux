토큰 사용량

상태줄의 토큰 배지를 클릭하면 사용량 팝업이 열립니다. 화면 스크랩이 아니라 ~/.claude/*.jsonl 트랜스크립트 회계를 권위 출처로 삼아 cache_read 까지 정확히 집계합니다(스크랩 누계는 실제의 약 0.4% 에 불과합니다).

여는 법과 탭 이동

상태줄 토큰 배지 클릭 또는 token-log(별칭 token-usage)로 엽니다. 기간 / 세션 / 계정 / 한도 노트북 탭으로 구성되고, 상단 요약줄(5h% · 주% · ~Σ)은 모든 탭에 공통으로 붙습니다. 탭 사이는 키로도 오갑니다 — p(세션) · l(한도) · u(/usage).

기간 뷰 — 시간 흐름으로

토큰을 월 → 주 → 일 → 시각 계층 트리로 봅니다. 각 행에 토큰 합과 가로 막대가 붙고, Enter / ←→ 로 주를 일·시각까지 펼치고 접습니다. 언제 토큰이 몰렸는지 시간 축으로 짚을 때 씁니다.

세션 뷰 — Claude 세션별로

p 로 전환합니다. 행이 Claude 세션별 합(대표 탭:패널 라벨 · 타임스탬프 · 토큰)입니다. 닫히고 재사용되는 패널 id 대신 안정적인 세션 id 로 묶어, 어느 세션이 토큰을 많이 썼는지 비교할 때 씁니다. 계정 뷰로는 여러 계정을 나눠 볼 수 있고, 필요하면 token-account 로 현재 패널의 계정을 직접 지정합니다.

한도 뷰 — /usage 실측과 리셋

l 로 전환합니다. /usage 실측 한도(세션 5h · 주 전체 · 주 Sonnet)를 % 사용 막대와 리셋 시각으로 보여주고, 현재 5h 창 추정 Σ 와 다음 리셋까지를 블록 숫자 카운트다운 시계로 보여줍니다(매 초 갱신). 첫 두 행에서 ←→ 로 모델·컨텍스트를 바꿔 적용할 수도 있어, 한도를 보며 모델을 고를 때 씁니다.

실측 한도는 숨은 세션으로 /usage 를 조회하는 claude-usage(별칭 usage)로 갱신되고, 막대만 따로 보려면 usage-panel(별칭 limits)을 씁니다. 다음 리셋까지의 카운트다운만 크게 보려면 usage-view([popup|tab|pane])를 씁니다.

경고 배지

Claude 가 이상하게 돌 때 상태줄에 경고 배지가 뜹니다 — "같은 출력 N회 반복 — 루프 의심", "Claude 포맷 미인식 — 추적 중단" 등. 팝업에는 이런 경고 이력도 함께 남습니다. 토큰 회계 자체를 진단하려면 token-debug(기본 off) 로그를 켤 수 있습니다.
