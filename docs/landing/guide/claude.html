Claude Code 연동

패널에서 claude CLI 를 돌리면 pytmux 가 상태·토큰·한도를 추적하고, 리밋에 걸리면 자동으로 잇습니다. 이 모든 기능은 claude-code 플러그인이며, 디렉토리를 지우면 조용히 사라집니다.

상태 아이콘과 알림

실행 중인 탭에 상태 아이콘을 표시합니다 — 대기 ○ · 처리중 ◐ · 리밋 멈춤 ⊘. 비활성 탭의 작업이 끝나면 탭 배경색으로 알립니다. 상태줄에는 모델 배지·토큰 배지 등 클릭 가능한 배지가 붙습니다.

토큰 리밋 자동 재개

사용량 리밋에 걸려 멈추면 출력의 해제 시각을 읽어 그때 자동으로 재개합니다. prefix R(또는 auto-resume on)로 토글하며 상태줄에 AR 로 표시됩니다. 재개 시 보낼 메시지는 auto-resume-message 로 바꿀 수 있습니다. 전송 오류(API 오류·레이트 리밋)에는 auto-retry(기본 on)가 1분 뒤 "continue" 를 넣어 이어 갑니다.

권한 모드 · 모델 · 시작 규칙

패널의 권한모드 footer 를 클릭하면 auto / default / plan 선택 팝업이 열립니다. 모델 배지 를 클릭하면 모델·컨텍스트를 바꾸는 팝업이 열립니다(내부적으로 /model 주입). claude-auto-mode 로 Claude 가 idle 일 때 자동으로 auto 모드로 바꾸게 할 수도 있습니다.

claude-rules 로 시작 규칙을 정의해 두면 새 세션이 시작되거나 /clear 뒤에 자동으로 주입됩니다. 새 세션에서 /rc(원격 제어)와 권한 auto 를 한 번 자동 적용하는 auto-launch(기본 on)도 있습니다.

프롬프트 히스토리

보낸 프롬프트 이력을 미리보기·검색·점프합니다(claude-prompt-history 플러그인, prompt-history·별칭 ph). 명령 프롬프트에서 입력하는 동안 직전 프롬프트를 미리 보여 주고, 미리보기 줄 수는 prompt-history-lines(1–3)로 조정합니다.

화면 안정성 (CJK 모호폭 · 깨짐 완화)

Claude Code 처럼 부분 갱신이 많은 TUI 는 단말마다 다른 모호폭(ambiguous-width) 처리 때문에 한글·기호가 겹쳐 보일 수 있습니다. pytmux 는 시작 시 폭을 자동 감지하고, 런타임에 :set ambiguous-width narrow|wide|auto 로 폭 모델을 일관되게 전환합니다.

드물게 출력이 어긋나면 claude-auto-redraw(off|idle|corruption, 기본 off)로 완료(busy→idle)마다 또는 깨짐 감지 시 패널을 한 번 전체 재출력해 자동 복원하게 켤 수 있습니다. 수동으로는 prefix r(redraw)로 언제든 다시 그립니다. 컨텍스트가 하드스톱에 닿으면 자동으로 /compact 를 부르는 자동 복구도 있습니다.

세션 이름 통일 · 세션 재개 · 피드백 숨김

  • 이름 통일 — namesync(claude-name-sync 플러그인)로 특정 디렉토리에서 Claude 를 돌리면 탭·패널·Claude 세션 이름을 키워드로 자동 통일합니다.
  • 세션 재개 — claude-resume(별칭 cr)로 이 머신의 Claude 세션 목록을 보고 Enter 로 새 탭에서 재개합니다.
  • 피드백 숨김 — claude-disable-feedback 플러그인이 Claude 의 피드백 유도 문구를 숨깁니다(별도 명령 없음).
  • 세션당 클리어 — prompt-clear 로 매 완료마다 문서화 후 /clear 하는 모드를 켤 수 있습니다.
토큰 사용량을 정확히 집계하는 팝업은 분량이 많아 다음 챕터 9. 토큰 사용량에서 따로 다룹니다.
