## 0. 핵심 결론(먼저)

지금 화면은 **텍스트를 한 줄씩 이어 붙인** 목록이라(`usagelog.summary_lines`
`usagelog.py:152-180`), ① 모바일 좁은 폭에서 한 항목이 2~3줄로 **접히고**, ②
숫자가 **정렬되지 않아** 크기 비교가 안 되며, ③ 계정이 하나뿐이어도 버킷마다
계정 합계를 **중복**으로 적고, ④ 막대 같은 **시각적 크기감**이 없다. 데이터·집계는
이미 정확하므로(58.4M 정확 집계), 개선은 **순수 표시 계층**(summary_lines 출력
형식 + TokenLogScreen 헤더/탭)만 손대면 된다 — 저장·집계·IPC 변경 0.

권고: **4단계**(§5). M1(컬럼 정렬 + 단일그룹 중복 제거)만으로도 체감 가독성이
크게 오르고 위험이 가장 낮다.

## 1. 현재 화면의 문제(스크린샷 + 코드 근거)

실제 화면(2026-06-07, 모바일 폭):
```
── 토큰 로그 · 계정: me@woojinkim.org
· 묶음: 계정 (탭/키 h·d·w·m, [a]계정,
[p]패널) ──
토큰 사용량 – 단위:day · 계정별 ·
계정=me@woojinkim.org  전체 Σ51.2M
  [me@woojinkim.org] Σ51.2M

2026-06-07  Σ327k
me@woojinkim.org:327k
2026-06-06  Σ6k    me@woojinkim.org:6k
2026-06-04  Σ50.9M
me@woojinkim.org:50.9M
```

| # | 문제 | 근거 | 영향 |
|---|---|---|---|
| P1 | **줄바꿈 과다** — 헤더·요약·버킷 한 항목이 좁은 폭에서 2~3줄로 접힘 | `_refresh` 헤더 한 줄에 안내문구까지 합침(`clientscreens.py` _refresh); 버킷 줄 `{bk} Σ… {parts}`(`usagelog.py:173-179`) | 한 화면에 몇 줄 못 봄, 시선 끊김 |

| P3 | **단일 그룹 중복** — 계정/세션이 하나면 `2026-06-07 Σ327k me@…:327k` 처럼 같은 수를 두 번 | 버킷 줄이 항상 그룹 분해를 덧붙임(`usagelog.py:174-178`) | 노이즈, 폭 낭비 |
| P4 | **시각적 크기감 없음** — 50.9M vs 327k 차이가 텍스트로만 | 막대/스파크라인 없음 | "어디에 몰렸나"가 즉시 안 보임 |

## 2. 목표 레이아웃(개선 후)

좁은 폭에서도 **한 항목 = 한 줄**, 숫자는 **우측 정렬**, 비율은 **막대**로:

```
 토큰 사용량 · 일 · 계정별            Σ51.2M     ← 제목줄(굵게)
 계정 me@woojinkim.org · 정렬 토큰↓           ← 스코프(옅게, 1줄)
─────────────────────────────────────
 me@woojinkim.org            51.2M ████████ 100%   ← 그룹 총합(막대)
─────────────────────────────────────
 06-07   327k  ▏                                   ← 버킷(우측정렬+막대)
 06-06     6k  ▏
 06-04  50.9M  ████████████████  99%
 h 시간 · d 일 · w 주 · m 월 · a 계정 · p 패널     ← 키 안내(맨 아래 1줄, 옅게)
```

설계 원칙:
- **한 항목 한 줄**: 폭을 넘으면 접지 말고 **줄임/생략**(막대·% 우선 희생). 안내문구는
  본문에서 빼 맨 아래 한 줄(footer)로.
- **우측 정렬 숫자**: `_fmt_tokens` 결과를 고정폭(예: 7칸)으로 우측 정렬 → 자릿수 비교.
- **막대 게이지**: 그 화면 최대값 대비 비율을 블록 문자(`█▉▊…▏`)로. 폭에 비례 길이.
- **단일 그룹이면 분해 생략**(P3): 그룹이 1개면 버킷 줄에 그룹명 반복 안 함.
- **반응형 폭**: 좁으면 막대·%·날짜 연도 순으로 떼고, 넓으면 모두 표시(`w` 기준 분기).

## 3. 표시 규약(구현 명세)

### 3.1 숫자·날짜 포맷
- `_fmt_tokens` 는 그대로(k/M). **고정폭 우측 정렬 헬퍼** 추가:
  `_rjust_tokens(n, width=7)` → `"  50.9M"`.
- 버킷 라벨 단축: day 는 `MM-DD`, week 는 `W##`, month 는 `YYYY-MM`, hour 는 `HH시`.
  연도는 제목/스코프에 한 번만(반복 제거).

### 3.2 막대 게이지
- `_bar(value, vmax, cells)` → 부분블록 포함 막대 문자열. `cells` 는 가용폭에서
  라벨·숫자·% 를 뺀 나머지(좁으면 0 → 막대 생략).
- 기준 vmax: **현재 화면(버킷 목록)의 최대 버킷 합**. 그룹 총합 줄은 전체 대비 %.

### 3.3 단일 그룹 처리(P3)
- `summary_lines` 가 그룹 수 1 이면: ① 상단 "그룹별 총합" 블록 생략(전체 Σ 와 동일),
  ② 버킷 줄에서 `{group}:{tok}` 분해 접미사 생략. 다중 그룹이면 현행처럼 상위 그룹
  몇 개만 접미사로(나머지는 `+N` 표기).

### 3.4 헤더/푸터 분리(P5)
- 제목줄: `토큰 사용량 · {버킷} · {묶음}별   Σ{전체}`(굵게).
- 스코프줄: `계정 {필터} · 정렬 {기준}`(옅게, 필터 없으면 `전체`).
- 키 안내: **본문에서 제거**하고 리스트 **맨 아래 1줄**(옅게) 또는 `?` 토글.

### 3.5 탭 줄 반응형(P6)
- 좁은 폭에서는 동작 탭(계정/패널/usage/시나리오)을 **아이콘/1글자**(계,패,U,S)로
  축약하거나, 버킷 탭(시/일/주/월)만 보이고 동작은 키(`a/p/u/s`)로. 모든 탭이 폭 안에
  들어오게 `w` 기준 라벨 길이 선택.

### 3.6 세션 식별 보강(P7) — [패널] 탭 한정
- `세션 4` → `세션 4 (탭2:p3)` 처럼 그 세션의 **대표 tab:pane**(레코드 최빈/최근)과,
  공간 되면 마지막 사용 시각을 곁들임. 라벨은 `usagelog` 가 records 에서 유도.

## 4. 정렬·접기(스캔성)
- **정렬 토글**: 토큰 많은 순(기본) ↔ 시간 순. 키 `o`(order) 또는 제목 클릭.
- **상위 N + 기타 접기**: 그룹(특히 세션)이 많으면 상위 N(예 8)만 막대로, 나머지는
  `기타 N개 Σ…` 한 줄로 접어 길이 폭주를 막는다(silent 절단 아님 — 접힘을 명시).

## 5. 단계(권고)
- **M1(즉시·저위험)** — 컬럼 우측정렬(P2) + 단일 그룹 중복 제거(P3) + 헤더/안내 분리
  (P5). `summary_lines` 출력 형식만 손봄. 막대 없이도 가독성 크게 향상.
- **M2** — 막대 게이지(P4) + 반응형 폭(P1) — 가용폭 계산은 화면 폭(`w`)을 인자로.
- **M3** — 탭 줄 반응형 축약(P6) + 키 안내 footer 토글.
- **M4** — 세션 라벨 보강(P7) + 정렬 토글·상위N 접기(§4).

각 단계 독립 커밋·테스트. 검증 게이트: 좁은 폭(예 40·60칸)에서 **한 항목 한 줄** 유지,
숫자 우측정렬 자릿수 일치, 단일 그룹 시 분해 미표시, 합계 보존(표시 ≠ 집계 왜곡).

## 6. 위험·완화
- **폭 계산 오차**: 한글(2셀)·이모지 폭. 기존 `_char_cells`(clientutil) 재사용해 셀폭
  기준으로 자른다(상태줄과 동일 규약).
- **막대가 정보를 가림**: 막대는 보조 — 숫자를 우선 배치하고 막대는 남는 폭에만.
- **순수성 유지**: `summary_lines` 는 폭(`width`)을 인자로 받는 **순수 함수**로 유지해
  테스트가 폭별 출력으로 단언 가능하게 한다(현 테스트 패턴 계승).

## 7. 미해결/후속
- 막대 기준(화면 최대 vs 전역 최대)·정렬 기본값은 사용성 보고 조정.
- `/usage`(M19 실측 한도)는 본 화면 상단에 별도 — 본 시나리오 범위 밖(표시 일관성만 맞춤).
- 컬럼 표를 Textual `DataTable` 로 바꿀지(정렬·스크롤 내장) vs 현 ListView 유지 — M2
  때 폭 제어/모바일 동작 보고 결정.
